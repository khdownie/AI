# Connect: Looking Back

**Ken Downie | October 1, 2025 – March 31, 2026**

## Deliver secure, high-quality content for customers

Drove a measurable reduction in Azure Files identity-related support cases by overhauling the identity-based authentication documentation for both Windows and Linux. Identity has long been the top driver of Azure Files support volume due to the inherent complexity of Active Directory and Entra ID integration. Over the last four full months (December 2025–March 2026), eligible support sessions declined from 18,000 to 10,900 — a 39% drop — while successful deflections followed a similar downward trend (6,500 to 4,400), indicating that fewer customers are encountering identity problems in the first place. This overhaul simplified sprawling content, removed outdated procedures, edited aggressively for brevity, and reorganized the identity doc set so customers can find the right guidance faster. The work also encompassed removing RC4 references from all identity docs to align with current security best practices, clarifying WinHTTP service requirements for Entra Kerberos, adding a new article on how to change Active Directory sources for identity-based authentication, and documenting managed identity property portal experiences — all reinforcing a more secure and intuitive authentication story.

Shipped Entra Kerberos cloud-only identity support documentation for Ignite 2025, enabling customers to authenticate to Azure Files SMB shares using cloud-only Entra ID identities without on-premises AD infrastructure. Coordinated closely with Ryan Thompson, Ross McAllister, and Hussein Al Kazwini to resolve build issues, merge conflicts, and unblock release PRs under tight Ignite timelines. This feature documentation expanded Azure Files' addressable audience to organizations operating entirely in the cloud and shipped on schedule alongside the product announcement.

Published managed identities support for Azure Files on Linux (public preview), giving customers a passwordless, certificate-free option for mounting SMB file shares from Linux workloads. Authored documentation for both system-assigned and user-assigned managed identity configurations, including the portal experience, and added macOS Platform SSO guidance for Entra Kerberos authentication. These efforts advance Azure Files' security posture by reducing reliance on stored credentials and broadening cross-platform support.

Authored a new Azure Files throttling article to publicly document throttling limits and behaviors that were previously undocumented. Multiple customers had been surprised by unexpected throttling, generating frustration and support escalations. The new content helps customers proactively design around throttling limits, set appropriate expectations, and troubleshoot throttling-related performance issues without filing support tickets.

Improved the Troubleshoot ClientOtherErrors article after multiple support engineers cited these errors as a persistent source of customer confusion and ticket volume. Updated the content to help both support engineers and customers interpret storage analytics logs and resolve issues faster, reducing mean-time-to-resolution for a common class of support interactions.

Delivered Robocopy migration content improvements by reworking guidance that had become outdated and was generating support cases from customers encountering issues during SMB migrations. The refreshed content provides clearer recommendations and updated best practices, contributing to smoother migrations and fewer escalations.

Completed a quality pass on classic vs. managed file shares (Microsoft.Fileshares) preview content introduced by PMs. Since the new top-level resource provider changes how file shares are deployed — starting with NFS and expanding to SMB later in 2026 — clear, accurate documentation is essential to avoid customer confusion, make working with Azure Files more intuitive, and support service adoption as the feature progresses toward general availability.

Published zone pinning and zonal co-location documentation, the Windows SMB permission model including admin/take ownership at GA, and Network Security Perimeter (NSP) documentation for Azure Files. Each of these features addresses specific customer needs around availability, security, and access control, and each required coordination with PM partners to ensure accuracy and completeness. Also documented a workaround for Linux SMB mounts failing with AES-256-GCM encryption and additional causes for Error 67, addressing active customer pain points surfaced through support channels.
